2017微软预科生面试

一天面试了三轮

今年面试的超多人， 第一波9点开始，一般都会面两轮，两轮之后会刷一部分人

前两轮面试基本上就是简历上的项目+算法题目

  • 第一轮面试题目给定一个数组，找到子序列和为奇数的最大值

    上来直接就写dp状态转移方程式，被面试官一个反例直接推翻，突然想起来

    和是奇数 = 前边的序列和是偶数 + num[i] 是奇数

    和是偶数 = 前边的序列和是奇数 + num[i] 是偶数

    那么需要保存的就是偶数以及奇数两种情况

    类似求一个数组中子序列乘积最大的求法

    写完之后，第一个点考虑溢出的情况，我说把保存结果的两个数组改为long

    ​ 第二个点考虑算法的复用性，一直在考虑max算法的重载，表示有点晕，然后面试官提醒说把max,min作为参数传进来

    ​

  • 第二轮面试题目

    • 给定两个数组，一个递增，一个递减，找到两个数组重合的数

      从O(nlogn) 到 O(n)的改进

      这个两个端指针分别遍历，实现不是很难

      当时体力不支，面试官开始问时间复杂度的时候，已经有点蒙了， 最好O(min(m,n)) ,最坏 O((m + n))

      但其实，我脑袋里的想法都是线性的，并没有看到很不一样的地方

    • 给一个字符串求全排列

      表示这个递归真心写的不爽，不过微软的工程师都很nice，在你不想写的时候，他们会鼓励你继续写，我觉得这个是超吸引的一点，能学到很多，你会知道自己当前的几种思路哪些不可行，他们对与方法可行性的判断表示更迅速准确，面试是个跟他们有互动的良好机会，能学到很多

      ​

  • 第三轮面试

    ​ 遇到一位小帅锅，蹭了顿饭，错过了盒饭，微软食堂真心点赞

    ​ 唉，就是吃太饱了，整个人昏昏欲睡的

    ​ 面试官气场超强，关于项目的考察简直了，明明接触数据的是我，整个面下来，感觉是他给我讲方法，一个问题比一个问题犀利，问完之后，觉得自己之前什么都没做，嗯，没错就是这种感觉，不过现在想起来，面试官教给我的是解决问题的主观能动性，即使你现在的业务需求并没有这些，但是你的直觉以及兴趣会有更多可以做的东西。

    ​ 算法题目：给定两个有序数组，求其中第k个数

    ​ 我问数组有多大，他说可以接收，我就开始狂写了一个线性的算法，表示当时面试官在边上狂写代码，尴尬癌发作：

    ​ 面试官一抬头，“你这边界有问题吧？” … 是哦，一顿改

    ​ 面试官二抬头，“你这可能出现死循环吧？” …是哦，一顿改

    ​ 面试官三抬头，”你这算法的复杂度？”… 线性的… “怎么优化？”

    ​ “从一半开始找吧，莫名的写了2/k, 明明想写 k/2的” 面试官说:”别紧张吖，算法思路没错，接下来怎么办” ….

    ​ “…” 基本上就完全断片了

    ​ 面不下去了，面试官友情提醒我，编码能力有待提高，感觉gg了

    ​

    面了三面之后的结论是编码能力不太够，回去继续练习，估计gg了

    ​

    ps,面试过程中有面到大数加法的，也有面到股票的变化的，也有链表的深复制的，矩阵的运算，希望多多准备吧。
