搭建博客

2016年6月底用hexo + github 搭建了一个博客

然后来随着电脑的hexo卸载，这个博客就没有更新

今年3月份结束了一份实习之后，读了《构建之法》和《暗时间》之后，觉得博客这个东西还是得有一个的，就像我在看到一个学长在网站上传了自己高中的手写日记，突然像被击中了一样，尤其是我现在很多东西记不住的情况下

这个坑也就从这里埋下来了

很兴奋的去阿里云上买了一个域名

接下来就是买服务器，服务器哪里买呢？ 看网上推荐bluehost，然先后在美国官网以及香港官网上都购买了一台，不得不说，服务还是极好的，只可惜，访问速度那是一个慢，而且整个网站的wordpress那是一个盛行，其实我只是想搭个python，用下flask or django，折腾了几天之后，发现系统自带的python2.6更新到2.7好捉急

弃坑

外国的速度不满意，那就折腾回来吧，在阿里云上买了一个学生机器，刚绑定好域名和服务器，还沉浸在访问的欣喜中，结果一个“您的网站还未备案”把我气晕了，整个流程下来，工作日20天，也就是下个月才可以看到，但是想起访问的哗哗的速度，我决定还是乖乖备案，结果，结果，说是vip的域名北京不支持，如果想用服务器，要么换域名，要么通信地址换，于是，那就换通信地址吧，结果我的老家地址不支持我，唉，都是当初埋下的坑

没办法

那就先整个静态的网站吧，于是就回到了github上

然之前写的文章还在，我不想重新折腾，把这部分资源给搞没了，懒懒的拖了好几天

终于，我还是亲手把之前写的东西抹杀掉了，不过我一直在想，第一，如果我当时备注了那些.md文件是不是会好些？第二，我是不是在hexo的配置文件中的branch不写master会好点？

于是我在知乎上第一次提问题，不知道有人回答没

突然看上面的文字，觉得应该写个流程图，23333

简单的记录下整个过程：

  1. 下载 nodejs + github
  2. ssh 连接 本地github && 线上github
  3. 在线上创建github的仓库
  4. 绑定域名，这里，因为hexo每次都运行，所以要把CNAME的文件放到source下
  5. 找个自己满意的hexo主题，安装 [通过查看一百多个主题，如果有时间，我打算自己做一个，强迫症]
  6. github是.md格式文件，试了几款，推荐Typora
  7. 至此，就可以开始写文章
